# 逐步详细解析
# 第1步：激活近战攻击GA（GameplayAbility）
```
玩家输入 → ASC::TryActivateAbility(MeleeAbilityClass)
  → GA::CanActivateAbility() 检查:
      - CooldownTags 冷却检查
      - CostGE 消耗检查（如体力）
      - ActivationBlockedTags / ActivationRequiredTags
  → GA::ActivateAbility()
```
关键点：

GA 的 NetExecutionPolicy 设为 LocalPredicted，客户端预测激活，服务器验证

InstancingPolicy 建议 InstancedPerActor（近战攻击频繁复用，避免每次创建实例）

CommitAbility() 在此阶段调用，先扣Cost再扣Cooldown（注意二者非原子操作，商业项目需防护）

# 第2步：播放攻击蒙太奇（AbilityTask_PlayMontageAndWait）
```
GA::ActivateAbility()
  → UAbilityTask_PlayMontageAndWait::CreatePlayMontageAndWaitProxy()
      - MontageToPlay: 攻击蒙太奇资产
      - PlayRate: 攻速倍率（可从Attribute读取）
      - StartSection: 连招段数对应的Section名
  → Task->ReadyForActivation()
  → AnimInstance->Montage_Play(Montage, Rate)
```
关键点：

Task 绑定四个回调：OnCompleted、OnBlendOut、OnInterrupted、OnCancelled

蒙太奇播放后，动画蓝图开始Tick，进入攻击动画帧序列

网络同步：LocalPredicted 模式下客户端立即播放，服务器通过 AbilitySystemComponent::ServerSetReplicatedEvent 同步确认

# 第3步：AnimNotifyState 开启攻击检测窗口
```
动画时间轴到达 MeleeWindow 起始帧
  → UAnimNotifyState_MeleeWindow::NotifyBegin()
      - 通知GA或独立组件：开启碰撞检测
      - 清空本次攻击已命中列表 (HitActors TSet)
```
关键点：

AnimNotifyState（状态型通知）而非 AnimNotify（点型通知），因为近战检测是一个持续的时间窗口

NotifyTick() 每帧执行检测，NotifyEnd() 关闭窗口

商业项目推荐方式：NotifyState 通过 GetOwningAbility() 或 GameplayTag 通知到 GA/独立MeleeComponent，而非直接在Notify中写检测逻辑（解耦）

# 第4步：碰撞检测 — 每帧Sweep/Overlap判定命中
```
NotifyTick() 每帧调用
  → MeleeComponent::PerformMeleeTrace()
      方式A: UKismetSystemLibrary::SphereTraceMulti() — 武器骨骼插槽连线Sweep
      方式B: UWorld::OverlapMultiByChannel() — 扇形/胶囊体区域检测
  → 命中结果 FHitResult[]
  → 过滤:
      - 排除自身
      - 排除已命中目标 (HitActors TSet 去重，防止同一挥砍多次命中)
      - 排除友方 (IGenericTeamAgentInterface / TeamTag)
      - 排除无ASC的Actor
  → 命中确认 → 回调 GA::OnMeleeHit(TargetActor, HitResult)
```
关键点：

武器拖尾Sweep（商业级最佳实践）：在武器骨骼上设置 Base 和 Tip 两个 Socket，每帧记录上一帧位置，连线做 SphereTraceMulti，形成扫掠体积，解决高帧率下武器穿透问题

检测在服务器执行（权威），客户端仅做预测表现

TraceChannel 使用自定义碰撞通道（如 ECC_MeleeTrace），不与其他物理查询冲突

# 第5步：应用伤害GE到目标
```
GA::OnMeleeHit(TargetActor, HitResult)
  → 构造 FGameplayEffectSpecHandle:
      GE_Damage = MakeOutgoingGameplayEffectSpec(DamageGEClass)
  → 设置伤害参数 (SetByCaller):
      GE_Damage.Data.SetSetByCallerMagnitude(DamageTag, CalculatedDamage)
  → 构造 FGameplayEffectContext:
      Context->AddInstigator(OwnerActor, AvatarActor)
      Context->AddHitResult(HitResult)  // 命中信息（位置、法线、骨骼）
  → ASC->ApplyGameplayEffectSpecToTarget(GE_Damage, TargetASC)
```
关键点：

SetByCaller 传递动态伤害值，而非硬编码在GE Modifier中，实现一个GE复用于不同伤害技能

FGameplayEffectContext 携带 HitResult 是关键！后续受击方向判定、受击蒙太奇选择都依赖这个数据

商业项目建议自定义 FGameplayEffectContext 子类，扩展字段如：bIsCritical、DamageType、HitBone、KnockbackDirection 等（需重写 NetSerialize、Duplicate、GetScriptStruct 三个函数，否则网络序列化会静默丢失数据）

# 第6步：GE 应用到目标 ASC
```
TargetASC::ApplyGameplayEffectSpecToSelf()
  → 免疫检查 (Immunity Tags)
  → ApplicationRequirement 检查
  → 创建 FActiveGameplayEffect
  → 执行 Modifiers:
      - Damage GE 通常是 Instant 类型
      - Modifier: Attribute = Health, Op = Additive, Magnitude = -DamageValue
  → 触发 Aggregator 计算管线
```
关键点：

如果使用 ExecutionCalculation (ExecCalc)，伤害公式在这一步执行：读取攻击者Attack、目标Defense等多属性，计算最终伤害

Instant GE 直接修改 BaseValue，不进入持续效果管线

此步骤在服务器权威执行

# 第7步：AttributeSet 接收属性变更
```
UAttributeSet::PostGameplayEffectExecute(const FGameplayEffectModCallbackData& Data)
  → 获取伤害值: float Damage = Data.EvaluatedData.Magnitude
  → 伤害钳制: Damage = FMath::Max(Damage, 0.f)
  → 应用到Health: SetHealth(FMath::Clamp(GetHealth() - Damage, 0, GetMaxHealth()))
```
关键点：

PostGameplayEffectExecute 是伤害结算的核心枢纽，商业项目的伤害后处理全在这里

这里可以做：护盾吸收、伤害减免、死亡判定、伤害数字UI事件等

# 第8步：伤害结算后 → 发送受击GameplayEvent
```
UAttributeSet::PostGameplayEffectExecute()
  → if (Damage > 0.f)  // 实际造成了伤害
  {
      // 构造事件数据
      FGameplayEventData EventData;
      EventData.EventTag = Tag_GameplayEvent_HitReact;
      EventData.Instigator = Data.EffectSpec.GetContext().GetInstigator();
      EventData.Target = GetOwningActor();
      EventData.ContextHandle = Data.EffectSpec.GetContext(); // 携带HitResult!
      EventData.EventMagnitude = Damage; // 伤害值，用于判断轻重受击
      
      // 向目标ASC发送事件
      TargetASC->HandleGameplayEvent(EventData.EventTag, &EventData);
  }
```
关键点：

使用 GameplayEvent 驱动受击（而非在GA/GE中直接PlayMontage），这是商业项目最佳解耦方式

事件中携带 ContextHandle（含 HitResult），受击GA 可据此判断：

攻击方向（正面/背面/左/右）→ 选择对应方向的受击蒙太奇

命中骨骼 → 头部受击 vs 身体受击

伤害量级 → 轻受击 vs 重受击 vs 击飞

# 第9步：同时触发 GameplayCue（打击反馈）
```
// 方式A：GE自带Cue（配置在GE资产的GameplayCues数组中）
GE_Damage 的 GameplayCue Tag: GameplayCue.Combat.Hit.Melee
  → UGameplayCueManager::HandleGameplayCue()
      → 客户端执行: 播放打击特效、音效、镜头震动

// 方式B：手动触发（更灵活，商业项目推荐）
TargetASC->ExecuteGameplayCue(Tag_GC_MeleeHit, CueParameters);
  - CueParameters 包含: HitLocation, HitNormal, PhysicalMaterial 等
```
关键点：

GameplayCue 只在客户端执行表现，不影响游戏逻辑

推荐使用 Non-Replicated GameplayCue：服务器不发送Cue RPC，由客户端在预测阶段自行触发，减少网络带宽

打击停顿（HitStop）也在此环节实现：UGameplayStatics::SetGlobalTimeDilation 或 CustomTimeDilation

# 第10步：受击 GA 被 GameplayEvent 触发激活
```
// 目标角色预先通过 ASC->GiveAbility() 授予了 GA_HitReact
// GA_HitReact 配置了 AbilityTriggers:
//   - TriggerTag: GameplayEvent.HitReact
//   - TriggerSource: GameplayEvent

TargetASC->HandleGameplayEvent(Tag_HitReact, &EventData)
  → 遍历已注册的 Triggered Abilities
  → 找到 GA_HitReact，Tag 匹配
  → TryActivateAbility(GA_HitReact)
  → GA_HitReact::ActivateAbility(Handle, ActorInfo, ActivationInfo)
      // EventData 通过 ActivationInfo 传入
```
关键点：

受击GA使用 AbilityTrigger + GameplayEvent 被动激活，不走输入系统

NetExecutionPolicy 设为 ServerInitiated（服务器发起，因为伤害结算在服务器）

可以设置 ActivationBlockedTags：如 State.Immune、State.SuperArmor 阻止受击（霸体机制）

# 第11步：受击 GA 内部 — 选择并播放受击蒙太奇
```
GA_HitReact::ActivateAbility()
{
    // 1. 从EventData提取命中信息
    FGameplayEffectContext* Context = EventData.ContextHandle.Get();
    const FHitResult* HitResult = Context->GetHitResult();
    float DamageMagnitude = EventData.EventMagnitude;
    
    // 2. 计算攻击方向
    FVector HitDirection = (HitResult->TraceEnd - HitResult->TraceStart).GetSafeNormal();
    FVector ForwardVec = GetAvatarActor()->GetActorForwardVector();
    float DotProduct = FVector::DotProduct(ForwardVec, HitDirection);
    float CrossZ = FVector::CrossProduct(ForwardVec, HitDirection).Z;
    
    EHitReactDirection Direction;
    if (DotProduct > 0.707f)        Direction = Front;    // 正面受击
    else if (DotProduct < -0.707f)  Direction = Back;     // 背面受击
    else if (CrossZ > 0)            Direction = Right;    // 右侧受击
    else                            Direction = Left;     // 左侧受击
    
    // 3. 判断受击等级
    EHitReactLevel Level;
    if (DamageMagnitude > HeavyThreshold)      Level = Heavy;   // 重受击
    else if (DamageMagnitude > LightThreshold)  Level = Light;   // 轻受击
    else                                        Level = Flinch;  // 微颤
    
    // 4. 查表选择蒙太奇 (DataAsset数据驱动)
    UAnimMontage* HitReactMontage = HitReactDataAsset->GetMontage(Direction, Level);
    
    // 5. 播放受击蒙太奇
    UAbilityTask_PlayMontageAndWait* MontageTask = 
        UAbilityTask_PlayMontageAndWait::CreatePlayMontageAndWaitProxy(
            this, NAME_None, HitReactMontage, 1.0f);
    MontageTask->OnCompleted.AddDynamic(this, &ThisClass::OnMontageEnded);
    MontageTask->OnInterrupted.AddDynamic(this, &ThisClass::OnMontageEnded);
    MontageTask->ReadyForActivation();
}
```
关键点：

受击蒙太奇选择使用 DataAsset 数据驱动：TMap<(Direction, Level), UAnimMontage*>，策划可配置

受击GA结束时清理：EndAbility() 自动清理 AbilityTask

如果受击过程中再次被命中：

InstancingPolicy = InstancedPerActor + 新的 TryActivateAbility 会打断当前蒙太奇、重新播放

或使用 bRetriggerInstancedAbility = true 重触发

# 第12步：蒙太奇播放 → 动画表现
```
UAbilityTask_PlayMontageAndWait::Activate()
  → UAnimInstance::Montage_Play(HitReactMontage)
      → 动画蓝图 AnimGraph 中 Montage Slot 接管骨骼
      → 混合入受击姿态
      → 蒙太奇播放完成/混出
  → Task::OnMontageBlendingOut / OnMontageEnded
  → GA_HitReact::EndAbility()
```
# 完整调用链一览（精简版）
```
玩家按键输入
  → ASC->TryActivateAbility(GA_MeleeAttack)
    → GA::ActivateAbility()
      → AbilityTask_PlayMontageAndWait (攻击蒙太奇)
        → AnimNotifyState_MeleeWindow::NotifyTick()
          → SphereTraceMulti (武器骨骼Sweep)
            → 命中目标 → GA::OnMeleeHit()
              → MakeOutgoingGameplayEffectSpec(GE_Damage)
              → SetByCaller设置伤害值
              → ApplyGameplayEffectSpecToTarget(目标ASC)
                → [ExecCalc计算伤害公式]
                → AttributeSet::PostGameplayEffectExecute()
                  → Health减少
                  → HandleGameplayEvent(Tag.HitReact, EventData)
                    → GA_HitReact被动激活 (AbilityTrigger)
                      → 解析HitResult→方向/等级
                      → DataAsset查表→选受击蒙太奇
                      → AbilityTask_PlayMontageAndWait (受击蒙太奇)
                        → AnimInstance::Montage_Play()
                  → GameplayCue → 打击特效/音效/HitStop
```
# 商业项目关键最佳实践总结
环节	最佳实践	避坑
近战检测	武器骨骼Socket Sweep，TSet去重	不要用简单Overlap球，精度不够
伤害传递	SetByCaller + 自定义EffectContext	不扩展Context会丢失HitResult网络数据
受击触发	GameplayEvent驱动，不直接耦合	不要在攻击GA里直接调目标的PlayMontage
蒙太奇选择	DataAsset数据驱动，方向+等级查表	不要硬编码if-else选蒙太奇
打击反馈	Non-Replicated GameplayCue	减少不必要的服务器→客户端Cue RPC
网络同步	攻击GA LocalPredicted，受击GA ServerInitiated	检测在服务器权威执行，客户端做预测表现
霸体/免疫	ActivationBlockedTags + GE Immunity	霸体不是不受伤，是不播受击动画
